\subsection{Анализ производительности \texttt{HttpUnit}}

\subsubsection{Описание программы}

\texttt{HttpUnit} --- это Java-библиотека для автоматизации тестирования веб-сайтов через имитацию работы браузера на уровне протокола HTTP. Она позволяет без запуска реального браузера:
\begin{itemize}
  \item переходить по ссылкам;
  \item заполнять формы;
  \item проверять содержимое ответов.
\end{itemize}

Такой подход делает тесты быстрыми и легковесными. Чаще всего инструмент применяется вместе с JUnit для создания интеграционных тестов веб-интерфейсов.

Подробнее с проектом можно ознакомиться на внешнем ресурсе\cite{httpunit_website}.

\subsubsection{Узкие места в потоке \texttt{main}}

\subsubsubsection{Описание проблемы}

В процессе выполнения сценария в главном потоке наблюдается необоснованные задержки, которые приводят к деградации времени отклика приложения.

\subsubsubsection{Алгоритм локализации проблемы}

Для изоляции узкого места выполнялись следующие шаги:
\begin{enumerate}
    \item Запуск приложения под управлением профайлера (VisualVM / IntelliJ IDEA Profiler).
    \item Запись CPU Profile (или Sampling) в момент выполнения целевой операции.
    \item Анализ дерева вызовов (Call Tree) и выявление методов с наибольшим собственным временем выполнения (Self Time).
\end{enumerate}

\subsubsection{Избыточная активность сборщика мусора}

\subsubsubsection{Описание проблемы}

Наблюдается высокая частота вызова GC, что снижает общую пропускную способность системы из-за постоянного выделения краткоживущих объектов.

\subsubsubsection{Алгоритм локализации проблемы}

\begin{enumerate}
    \item Подключение вкладки \texttt{VisualGC} в VisualVM или мониторинг памяти в IDE.
    \item Фиксация пилообразного графика памяти в области Eden и частоты операций сборки.
    \item Использование режима "Memory Profiling" для определения классов, создающих наибольшую нагрузку на кучу (Heap Allocation Rate).
\end{enumerate}

\subsubsection{Рост потребления области Metaspace}

\subsubsubsection{Описание проблемы}

Постоянный тренд роста объема памяти, занимаемой областью Metaspace, что указывает на потенциальную утечку памяти, вызванную бесконтрольной динамической генерацией и загрузкой классов.

\subsubsubsection{Алгоритм локализации проблемы}

\begin{enumerate}
    \item Мониторинг области Metaspace в VisualVM на длительном промежутке времени.
    \item Снятие дампа кучи (Heap Dump) и его последующий анализ с помощью утилиты Eclipse Memory Analyzer (MAT) или встроенных средств IDE.
    \item Анализ загруженных классов (\texttt{Duplicate Classes} или поиск динамически создаваемых классов).
\end{enumerate}
