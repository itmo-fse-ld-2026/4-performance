\subsubsection{Листинг}

\begin{lstlisting}[caption={Часть содержимого файла \texttt{build.xml}},captionpos=b]
<target name="alt">
  <delete dir="${fse.alt.dir}" failonerror="false"/>
  <mkdir dir="${fse.alt.dir}"/>

  <echo message="Creating alternative version of project..." />

  <copy todir="${fse.alt.dir}">
    <fileset dir="${fse.compile.source}"/>
  </copy>

  <for list="${fse.alt.pairs}" param="pair" delimiter=",">
    <sequential>
      <local name="from"/>
      <local name="to"/>

      <propertyregex property="from"
                     input="@{pair}"
                     regexp="^(.+):"
                     select="\1" />
      <propertyregex property="to"
                     input="@{pair}"
                     regexp=":(.+)$"
                     select="\1" />
      <replaceregexp match="${from}" replace="${to}" flags="g" byline="true">
        <fileset dir="${fse.alt.dir}">
          <filename regex="${fse.alt.files}"/>
        </fileset>
      </replaceregexp>
    </sequential>
  </for>

  <for list="${fse.alt.pairs}" param="pair" delimiter=",">
    <sequential>
      <local name="from"/>
      <local name="to"/>

      <propertyregex property="from"
                     input="@{pair}"
                     regexp="^(.+):"
                     select="\1" />
      <propertyregex property="to"
                     input="@{pair}"
                     regexp=":(.+)$"
                     select="\1" />
      
      <move todir="${fse.alt.dir}">
        <fileset dir="${fse.alt.dir}">
          <include name="**/*"/>
        </fileset>
        <mapper type="regexp" from="^(.*)${from}(.*)$" to="\1${to}\2"/>
      </move>
    </sequential>
  </for>

  <antcall target="build">
    <param name="fse.compile.source" value="${fse.alt.dir}" />

    <param name="fse.build.version" value="${fse.build.version}-alt" />
  </antcall>
</target>
\end{lstlisting}

\subsubsection{Комментарии}

В начале работы цель пересоздаёт директорию альтернативной версии программы и копирует в неё исходный код текущей версии при помощи задач \texttt{delete}, \texttt{mkdir} и \texttt{copy} --- все доступны в составе Apache Ant API.

Далее происходит обработка заданных шаблонов замены из файла параметров. Они перечислены через запятую в формате:
\[
  \texttt{old\_substring:new\_substring}
\]

Чтобы обработать каждый шаблон в отдельности, воспользуемся задачей \texttt{for} из сторонней библиотеки \textit{Ant-Contrib} --- установим её через Apache Ivy. Задача \texttt{sequential} из Apache Ant API позволяет объединить несколько задач в одну в порядке их следования, чтобы \texttt{for} мог их исполнить \textit{(он не поддерживает несколько задач в теле)}.

В начале заменим все вхождения подстроки в отслеживаемых файлах при помощи задачи \texttt{replaceregexp}. Задача \texttt{propertyregex} позволяет записать в локальные переменные подстроки для поиска и замены. Упомянутые задачи входят в состав Apache Ant.

Затем аналогично поступим с файловым деревом, поскольку для компиляции необходимо, чтобы имена Java-классов совпадали с именами файлов. Для этого вместо \texttt{replaceregexp} воспользуемся \texttt{move} с \texttt{mapper} для перемещения файлов и директорий по шаблону \textit{(тоже входят в Apache Ant API)}.

Заметим, что свойства \texttt{from} и \texttt{to} объявлены при помощи задачи \texttt{local} из Apache Ant API. Она позволяет скрыть переменные для блоков выше, поскольку свойства можно инициализировать лишь один раз для конкретной цели. Если убрать эти строчки, то обработается максимум лишь первый шаблон замены в каждом из циклов.

Цель оснащена логгирующими сообщениями, которые реализованы с помощью задачи \texttt{echo} \textit{(доступна в составе Apache Ant API)}.
