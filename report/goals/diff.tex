\subsection{Цель \texttt{diff}}

\subsubsection{Листинг}

\begin{lstlisting}[caption={Часть содержимого файла \texttt{build.xml}},captionpos=b]
<target name="diff">
  <echo message="Checking for changes in target classes..." />

  <java classname="org.tmatesoft.svn.cli.SVN"
        classpathref="fse.ivy.path.ant"
        fork="true">
    <arg value="update"/>
  </java>

  <java classname="org.tmatesoft.svn.cli.SVN"
        outputproperty="fse.diff.out"
        classpathref="fse.ivy.path.ant"
        fork="true">
    <arg value="status"/>
  </java>

  <loadresource property="fse.diff.commitFiles">
    <string value="${fse.diff.out}"/>
    <filterchain>
      <linecontainsregexp>
        <regexp pattern="${fse.diff.classes}"/>
      </linecontainsregexp>
      <tokenfilter>
        <replaceregex pattern="^.[\s]+" replace="" flags="g"/>
      </tokenfilter>
      <striplinebreaks/>
      <trim/>
    </filterchain>
  </loadresource>

  <condition property="fse.diff.detected">
    <matches string="${fse.diff.out}" pattern="${fse.diff.classes}"/>
  </condition>

  <antcall target="-svn-commit-if-changed"/>
  <echo message="Done task 'diff'!" />
</target>

<target name="-svn-commit-if-changed" if="fse.diff.detected">
  <echo message="Target classes modified. Committing to SVN..."/>

  <tempfile property="fse.diff.targetsFile" 
            prefix=".commit-" 
            suffix=".txt"
            deleteonexit="true"/>
  
  <echo file="${fse.diff.targetsFile}" message="${fse.diff.commitFiles}"/>

  <java classname="org.tmatesoft.svn.cli.SVN"
        failonerror="true"
        classpathref="fse.ivy.path.ant"
        fork="true">
    <arg value="commit"/>
    <arg value="-m"/>
    <arg value="feat: changes in tracked classes (Ant-automated)"/>
    <arg value="--targets"/>
    <arg value="${fse.diff.targetsFile}"/>
  </java>
</target>
\end{lstlisting}

\subsubsection{Комментарии}

Для проверки наличия изменений в отслеживаемых файлах используется команда \texttt{svn status}. Она вызывается посредством Java-класса из сторонней библиотеки SVNKit \textit{(она добавлена при помощи Apache Ivy)}. Сам вызов происходит при помощи задачи \texttt{java} --- в ней установлен флаг \texttt{fork}, чтобы исполнять задачу вне текущей JVM во избежание возможных конфликтов.

Задача \texttt{loadresource} загружает в свойство содержимое текстового элемента --- вывод команды \texttt{svn status}. Он фильтруется посредством regex-выражений. Их реализует тип \texttt{filterchain}, который объединяет действие классов \texttt{linecontainsregexp} и \texttt{tokenfilter}. Все из них содержатся в Apache Ant API.

Если изменения обнаружены, исполняется вспомогательная цель \texttt{-svn-commit-if-changed}. Она вызывается при помощи задачи \texttt{antcall} --- тоже встроенной в Apache Ant API. Содержимое булевой переменной вычисляется при помощи задачи \texttt{condition}.

Во вспомогательной цели список отслеживаемых изменённых файлов записывается во временный файл при помощи задач \texttt{tempfile} и \texttt{echo} --- обе тоже содержатся в Apache Ant API.

Затем эти файлы добавляются в новую ревизию Subversion при помощи той же библиотеки SVNKit.

Обе упомянутые цели оснащены логгирующими сообщениями, которые реализованы также с помощью задачи \texttt{echo}.
