\subsection{Цель \texttt{music}}

\subsubsection{Листинг}

\begin{lstlisting}[caption={Часть содержимого файла \texttt{build.xml}},captionpos=b]
<target name="music">
  <sound>
    <success source="${fse.ant.resources}/sounds/build_success.wav" />
    <fail source="${fse.ant.resources}/sounds/build_fail.wav" />
  </sound>
  <antcall target="build" />
</target>
\end{lstlisting}

\subsubsection{Комментарии}

Для воспроизведения звуковых файлов используется задача \texttt{music}. Она входит в состав Apache Ant API и позволяет указать раличные файлы в случае успеха или неудачи сборки.

Задача не сработает, если указать зависимость цели от \texttt{build}, поскольку она видит ошибки лишь в теле текущей цели. Если ошибка случится в теле более ранней цели, то выполнение до \texttt{music} не дойдёт, и сборка упадёт с ошибкой без воспроизведение звуков.

Для решения этой проблемы достаточно вызвать цель \texttt{build} в теле текущей цели при помощи задачи \texttt{antcall} --- она также входит в состав Apache Ant API.

\subsubsection{Значения используемых параметров}

\begin{lstlisting}[language=Properties,caption={Часть содержимого файла \texttt{build.properties}},captionpos=b]
fse.ant.resources = src/main/resources/ant
\end{lstlisting}
